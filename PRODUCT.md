# Product

## Register

brand

## Users

Mid-to-senior Laravel developers already using or evaluating Inertia.js + React. They know Filament. They want the same "declare resources, get a full admin" experience but without Livewire — staying fully in React. Context: evaluating tools before starting a new project or rebuilding an admin panel.

## Product Purpose

Monorail is a server-driven UI framework for Laravel + Inertia.js + React. It lets developers declare resources, tables, forms, and filters in PHP — and get a fully rendered, type-safe React admin interface without writing frontend boilerplate. Success looks like a developer reading the homepage and immediately understanding: "This is the Filament for Inertia. I should install it today."

## Brand Personality

Precise, confident, pragmatic. A well-engineered tool that doesn't need to sell itself loudly. Authoritative without being cold. Like a senior engineer explaining something they built — no hype, just clarity.

Three words: **precise, confident, pragmatic**

## Anti-references

- Generic SaaS hero templates: big number metrics, gradient blobs, "10x your workflow" copy
- Over-animated marketing sites that feel more like pitch decks than developer tools
- No-code / low-code tool aesthetics (Bubble, Webflow, Glide) — Monorail is for real developers
- Anything that looks like a bootstrapped indie hacker launch page
- Heavy glassmorphism or neon-on-black aesthetics

## Design Principles

1. **Show, don't tell** — code speaks louder than copy. A real code example communicates more than three paragraphs of marketing.
2. **Developer confidence** — calm authority, never hyped. The tone is "here's what it does" not "you won't believe this."
3. **Familiar but distinct** — Filament users should feel at home with the mental model, but Monorail has its own visual identity.
4. **Precision over decoration** — every visual element earns its place. Decoration that doesn't clarify should be removed.
5. **The stack is the story** — Laravel → Inertia.js → React is the value chain. The design reinforces this progression.

## Accessibility & Inclusion

WCAG AA minimum. Standard developer-tool audience — no special accommodation constraints identified, but code blocks must be keyboard-navigable and color contrast must hold for both the dark landing page and light docs surface.
